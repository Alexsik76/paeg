# Симулятор Електронного Голосування (ПАЕГ)

Цей проект є симулятором систем електронного голосування, розробленим в рамках дисципліни ПАЕГ (Проектування архітектури електронного голосування). Застосунок дозволяє симулювати різні етапи та проблеми криптографічних протоколів голосування.

Вже реалізовано:

- **Лабораторна робота №1 (Простий протокол)**: Цифровий підпис (RSA) та шифрування відкритою експонентою для перевірки цілісності та автентичності.

## 🚀 Локальне розгортання та запуск

Для запуску цієї програми локально на вашому комп'ютері, виконайте наступні кроки (знадобиться Python версії 3.9 або новішої):

1. **Клонування або завантаження коду:** Відкрийте термінал у папці з проектом.
2. **Створення віртуального середовища:** Це ізолює бібліотеки цього проекту від вашої ОС.
   ```bash
   python -m venv .venv
   ```
3. **Активація віртуального середовища:**
   - **Для Windows (PowerShell):**
     ```powershell
     .\.venv\Scripts\Activate.ps1
     ```
   - **Для Windows (CMD):**
     ```cmd
     .\.venv\Scripts\activate.bat
     ```
   - **Для Linux/Mac:**
     ```bash
     source .venv/bin/activate
     ```
4. **Встановлення залежностей:**
   ```bash
   pip install -r requirements.txt
   ```
5. **Запуск застосунку:**
   ```bash
   streamlit run main.py
   ```
   Після виконання останньої команди, застосунок автоматично відкриється у вашому браузері за адресою `http://localhost:8501`.

## 📂 Структура проекту: Де що міститься?

Код розбитий на логічні модулі для легкого масштабування до наступних лабораторних робіт. Якщо вам потрібно знайти конкретну логіку, орієнтуйтесь на цей список:

- **`config.yaml`** — **Конфігурація (кількість виборців, імена кандидатів)**. Тут визначається, що система має 5 виборців та 2 кандидати за замовчуванням. Також тут описані доступні сценарії симуляції.
- **`core/crypto.py`** — **Криптографія та ключі**. У цьому файлі описано генерацію пар RSA-ключів, логік підписання ЕЦП та шифрування (гібридне шифрування Fernet + RSA).
  - **Чому гібридний протокол?** Чистий алгоритм RSA має жорсткі математичні обмеження на розмір даних, які він може зашифрувати (залежить від довжини ключа, зазвичай це близько 245 байт для 2048-бітного ключа). Оскільки наш цифровий бюлетень — це JSON-об'єкт, який містить сам голос (ID кандидата), ідентифікатор виборця та сам по собі громіздкий цифровий підпис `signature` (256 байт), він просто не поміщається в ці ліміти. Тому застосовується індустріальний стандарт гібридного шифрування: весь великий JSON шифрується швидким симетричним ключем Fernet (AES), а потім лише цей маленький симетричний ключ шифрується повільним асиметричним відкритим ключем RSA, що належить ЦВК.
- **`protocols/base.py`** — **Абстрактні шаблони (інтерфейси)**. Тут описані базові, порожні класи `BaseVoter` та `BaseCVK`, від яких мають успадковуватись майбутні лабораторні роботи.
- **`protocols/lab1_simple.py`** — **Властивості комісії, виборців і процедура голосування**.
  - Клас `SimpleVoter`: описує, як саме виборець хешує, підписує та шифрує свій голос перед відправкою.
  - Клас `SimpleCVK`: описує правила комісії (як вона розшифровує дані, перевіряє ЕЦП та зараховує бюлетені).
- **`protocols/scenarios.py`** — **Сценарії голосування**. Тут лежить логіка виконання різних симуляцій (подвійне голосування, успішні вибори, спроба підміни бюлетеня).
- **`main.py`** та **`ui/components.py`** — Графічний інтерфейс програми (кнопки, дашборди, меню вибору).
- **`core/i18n.py`** — Файл перекладів усіх текстових повідомлень системи на українську та англійську мови.

## 📝 Важливе зауваження щодо завдання

В методичних рекомендаціях є вимога:

> _"Врахуйте можливість однакової кількості голосів у кандидатів... програма має вивести повідомлення наприклад про потребу в повторному голосуванні"_.

Зверніть увагу: **для цієї умови не створено окремого фіксованого сценарію у випадному списку.**

Ця вимога реалізована глобально та автоматизовано. Програма постійно аналізує результати поточного підрахунку. Щоб продемонструвати цю поведінку:

1. Оберіть розрахунок нормального голосування (Нормальне голосування - 1 виборець).
2. Проголосуйте одним виборцем за першого кандидата.
3. Оберіть іншого виборця з випадаючого списку та проголосуйте за другого кандидата.

Як тільки на загальному табло рахунок стане рівним (наприклад, 1:1), система розпізнає це і знизу миттєво з'явиться відповідне попередження: **"Увага: Виявлено однакову кількість голосів. Потреба в повторному голосуванні"**.
